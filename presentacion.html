04 — Reglas de Negocio

Logica Inteligente

Tres reglas fundamentales que garantizan la integridad de cada reserva, complementadas con un flujo de aprobacion humana para decisiones criticas.

01
🚫
Sin Doble Reserva
El sistema valida en tiempo real que no existan reservas superpuestas para la misma sala, fecha y rango horario. Si hay conflicto, sugiere alternativas automaticamente.
02
👥
Validacion de Capacidad
Cada sala tiene una capacidad maxima definida. Si el numero de asistentes excede el limite, la reserva se bloquea y se ofrece una sala alternativa con capacidad suficiente.
03
⏱
Validacion de Horario
No se permiten reservas en el pasado ni con horario de fin anterior al de inicio. El sistema valida coherencia temporal de cada solicitud.

Human-in-the-Loop

Las salas con capacidad superior a 25 personas activan un flujo de aprobacion especial. La reserva queda en estado "pendiente" hasta que un Super Administrador la revise.

Este mecanismo garantiza supervision humana en decisiones de alto impacto, alineandose con las mejores practicas de IA responsable: la IA asiste, el humano decide.

El administrador puede aprobar o rechazar desde su panel, y el usuario recibe notificacion inmediata con el resultado, incluyendo un correo electronico generado por IA.

05 — IA Generativa

Emails inteligentes,
generados por IA

El sistema genera correos HTML personalizados para cada reserva, adaptando el contenido, tono y diseno segun el resultado (confirmacion o rechazo).

Variables del template

  • {nombre} Nombre del solicitante
  • {sala} Nombre de la sala reservada
  • {fecha} Fecha de la reserva
  • {horaInicio} Hora de inicio
  • {horaFin} Hora de finalizacion
  • {asistentes} Numero de personas
  • {motivo} Motivo de la reunion
  • {estado} Confirmada / Rechazada / Pendiente
  • {razon} Razon de rechazo (si aplica)
  • {alternativa} Sala alternativa sugerida

La IA genera el HTML completo del correo de forma dinamica, incluyendo estilos inline, iconografia y texto adaptado al contexto de cada reserva.

06 — Integracion

Webhook n8n

Bookspace se conecta con n8n a traves de un webhook para enviar correos electronicos reales. Una colaboracion con el equipo partner de automatizacion del Nodo.

webhook-payload.json JSON 
{
  "to": "usuario@empresa.com",
  "subject": "Reserva Confirmada - Sala Nebula",
  "htmlBody": "<html>...email IA...</html>",
  "reserva": {
    "sala": "Nebula",
    "fecha": "2026-03-14",
    "horaInicio": "10:00",
    "horaFin": "11:30",
    "asistentes": 8,
    "estado": "confirmada",
    "requiereAprobacion": false
  },
  "timestamp": "2026-03-14T10:00:00Z"
}
1

Bookspace genera reserva

El usuario completa el formulario y la validacion es exitosa.

2

IA genera email HTML

Se construye el correo con todas las variables de la reserva.

3

Webhook envia payload

POST al endpoint de n8n con el JSON completo de la reserva y el email.

4

n8n procesa y envia

El flujo de n8n recibe el payload, lo procesa y envia el correo real al usuario.

5

Usuario recibe confirmacion

Correo HTML profesional en la bandeja de entrada del solicitante.

Mejoras Futuras

Bookspace fue disenado para escalar. Estas son las mejoras planificadas para convertirlo en una solucion empresarial completa.

Migrar de localStorage a Supabase o Firebase para persistencia en la nube, sincronizacion multi-dispositivo y escalabilidad.

02

Login con Google o Microsoft para integracion con el ecosistema corporativo y gestion de identidad centralizada.

03

Alertas en tiempo real por Teams, Slack o notificaciones nativas del navegador para recordatorios y cambios.

Calendario visual

Vista tipo Google Calendar con drag-and-drop para reservar, reprogramar y visualizar la ocupacion semanal.

05

API REST

Endpoints documentados para integracion con ERPs, CRMs y otros sistemas empresariales de la organizacion.

Aplicacion nativa con React Native para reservar desde el celular con la misma experiencia de usuario.
